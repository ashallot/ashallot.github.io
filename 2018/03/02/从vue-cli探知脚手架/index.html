从vue-cli探知脚手架

发表于 2018-03-02

在有了迅捷开发的概念和需求后，cli随运而生，它将大大浓缩开发的时间与成本，在此，我将就vue-cli大致讲一下自动获取开发模板的脚手架的开发。

vue-cli源码

vue文件

vue文件是vue-cli的入口文件，在此处定义了该cli所具备的基本命令（-help -V除外）。

vue-init 文件

目前的需求是方便快捷的获取备好的模板进行开发，那么就init命令讲解。

download-git-repo是用于下载模板的插件，github上有其使用的方法，当然也可以查看源码。在下面将会用到这个插件下载public的资源。

generate是vue自己完成的一个js文件，主要作用是将下载下来的资源重命名并放入当前npm run的目录下。

program查看commander。

help就跳过不谈

setting

vue-init命令应该不陌生，使用方式：vue init webpack myproject
template 即webpack（来源于网上的开源模板，有cue团队维护），
rawName 即myproject（自己新建的项目名称），
name 即通过判定命令行中第二个参数是否存在所对应的值（路径或者参数），
to 即当前命令行运行的目录

通过判定第二个参数是否存在，使得用户能够选择是否在当前目录下创建新项目

此代码中，判定模板是否存在本地中，若是，则直接获取本地模板，反之从网络获取资源
officialTemplate 参数是对命令行参数的扩写，使其在download-git-repo调取资源时，符合预存模板的路径

在此函数中，通过调用download下载网络上的预存模板，temp为命令行运行的目录
通过调用generate将下载下来的模板重命名并创建至命令行当前目录

通过以上的讲解，不难通过修改vuecli的网络请求路径实现自己的简单cli（从网络拉取模板的cli）
